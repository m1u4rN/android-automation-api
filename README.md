# Android Automation API Microservice 📱🤖

Микросервис на базе **FastAPI** для оркестрации мобильных эмуляторов, взаимодействия с ADB, управления сервером Appium и выполнения UI-автотестов. 

Проект спроектирован с упором на безопасность, отказоустойчивость и готов к развертыванию через Docker.

---

## 🚀 Ключевые возможности (Features)

* **Управление эмуляторами (`routers/emulators.py`)**: 
  * Получение списка доступных AVD (Android Virtual Devices).
  * Получение статуса подключенных устройств.
  * Безопасное выполнение базовых ADB-команд.
* **Инфраструктура (`routers/infrastructure.py`)**: 
  * Запуск и остановка локального сервера Appium.
  * Установка и удаление Android-приложений (APK) на целевом устройстве.
* **UI-Действия (`routers/actions.py`)**: 
  * Выполнение пошаговых сценариев автоматизации (UI-тестов) через Pydantic-модели.
* **Безопасность (Security-First)**: 
  * Строгая валидация входных данных (использование безопасных Regex-паттернов) для защиты от уязвимостей класса **OS Command Injection**.
* **Отказоустойчивость**: 
  * Глобальные таймауты (10-60 секунд) для всех вызовов `subprocess` для предотвращения зависания (hanging) потоков.
* **Аудит (Audit Trail)**: 
  * Централизованное логирование всех действий и запросов для мониторинга работы сервиса.

---

## 🛠 Стек технологий

* **Язык**: Python 3.10+
* **Фреймворк**: FastAPI, Uvicorn, Pydantic
* **Инструменты тестирования**: Appium, Android SDK (ADB, Emulator)
* **Развертывание**: Docker, Docker Compose

🐳 Запуск проекта (через Docker)
Для запуска микросервиса на вашей машине должен быть установлен Docker Desktop (или Docker Engine + Docker Compose).

Склонируйте репозиторий и перейдите в папку проекта.

Соберите и запустите контейнер одной командой:

Bash
docker compose up --build
(Для запуска в фоновом режиме добавьте флаг -d: docker compose up -d --build)

Сервер запустится на порту 8000.

Для остановки сервера (если он запущен не в фоне) нажмите CTRL+C в терминале или выполните:

Bash
docker compose down
📖 Интерактивная документация (Swagger)
После запуска контейнера интерактивная документация ко всем эндпоинтам доступна прямо в браузере. Вы можете отправлять тестовые запросы без использования Postman.

Swagger UI: http://localhost:8000/docs

ReDoc: http://localhost:8000/redoc

🛡 Примечания для разработчиков
Логирование: В процессе работы микросервис генерирует файл app.log в корне проекта. Этот файл содержит важный журнал аудита (Audit Trail). Он добавлен в .gitignore и не отслеживается системой контроля версий в целях безопасности.

Зависимости окружения: При запуске вне Docker убедитесь, что в системе установлены переменные окружения ANDROID_HOME и пути к инструментам платформы (platform-tools), иначе вызовы ADB будут завершаться ошибкой. При использовании Docker-контейнера он использует сетевой режим host, чтобы иметь доступ к ADB на хост-машине.
